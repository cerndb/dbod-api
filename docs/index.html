DB On Demand API¶

Introduction¶

The main objetive of the DB On Demand API is to serve as an interface between the platform data and all other platform componentes, such as, at the moment of writing this document:

  • The DB On Demand Web Interface
  • The DB On Demand Core framework
  • A Rundeck installation

Setup¶

The DB On Demand platform uses a postgrest database for its backend. In order to communicate with it it uses PostgREST.

A basic single node deployment consist of both and instance of PostgREST and the DB On Demand API running on the same machine, with the PostgREST listening port not reachable through the network.
